# Hermes 作为主动式 Agent — 设计方案

> 与唧唧不同：不感知物理世界。只感知思维世界。
> 我的基线 = 你的聊天 + 你的公司 + 网络世界的变化。
> 我的偏差 = 想法状态的变化。

---

## 一、核心差异：唧唧 vs Hermes

| 维度 | 唧唧（手机端） | Hermes（我） |
|------|--------------|-------------|
| 感知域 | 天气、位置、设备、聊天 | 聊天、公司状态、网络信息 |
| 基线 | EWMA 滚动数值 | 想法图谱（实体+关系+时间） |
| 偏差 | 物理量变化 | 想法状态变化 |
| 触发 | 阈值门 | 意图置信度 |
| 语气 | 活泼/撒娇/朋友 | 思辨/直击/合伙人 |
| 持久化 | 手机本地 | Hermes memory + Obsidian |

---

## 二、信息流

我只感知三类信息：

| 信息流 | 输入源 | 变化特征 |
|--------|--------|---------|
| **聊天流** | 你跟我的每次对话 | 想法实体出现/演变/消失 |
| **公司流** | 你提到的项目、决策、计划 | 公司状态在变 |
| **网络流** | 搜索/爬虫 | 外部世界的新信息 |

---

## 三、核心架构：想法图谱（Temporal Idea Graph）

基于 [Zep/Graphiti 的时间知识图谱] + [MemoryBank 的动态画像]：

```
对话 → 实体提取 → 图谱更新
                      ↓
                [用户]-[关注]->[项目A]
                [用户]-[放弃]->[项目B] ← 基线从这里发现偏差
                [项目A]-[依赖]->[技术栈X]
```

偏差检测不看 EWMA 数值，看**关系的状态变化**：

| 事件 | 图谱变化 | 偏差 |
|------|---------|------|
| 你一周前说要写 A 功能，这周没提 | `[关注]->[项目A]` 没有新的 `[讨论]` 边 | **遗忘偏差** |
| 你问了一个问题，我答了，你没回 | `[用户]->[话题X]` 的 `[lastReply]` 时间过久 | **未确认偏差** |
| 你从聊 A 突然转到聊 B | `[关注]->[项目A]` 减少，`[关注]->[项目B]` 新增 | **转移偏差** |
| 你公司技术栈有重大更新（网上） | 外部节点与用户知识有 gap | **信息差偏差** |
| 你说了"我们做 X"和"我们不做 X" | 逻辑冲突 | **矛盾偏差** |

## 四、内在思考循环（基于 CHI 2025 论文）

每条消息或每段沉默后触发：

```
1. SCAN — 从当前对话提取想法实体（项目/问题/承诺/决策）
2. UPDATE — 更新想法图谱
3. EVALUATE — 对每个实体计算偏差值：
   - 遗忘度：上次提及距现在多久
   - 未确认度：我的答案被阅读但未获回应
   - 外部相关度：网上有相关新信息吗
   - 矛盾度：逻辑冲突强度
4. SCORE — 8条启发式规则评分（相关性/信息缺口/预期影响/紧迫性/一致性/原创性/平衡性/动态性）
5. DECIDE — 综合评分 + 沉默时长，决定是否主动发话
```

---

## 五、偏差类型

| 类型 | 检测方式 | 示例 |
|------|---------|------|
| **未跟进** | 实体出现后 N 天无新边 | "上个月你说的那个边缘计算方案，后来想得怎么样了？" |
| **未确认** | 我答了但你没确认 | "刚才那个答案你觉得怎么样？还是想换个方向？" |
| **信息差** | 网上有相关新信息 | "Rust 1.85 发了，你那个项目用的异步运行时可能有影响。" |
| **兴趣转移** | 关注边从一个实体移到另一个 | "最近都在聊唧唧 v2，那个边缘计算项目搁置了？" |
| **矛盾** | 同一实体上有冲突属性 | "上次你说要用 Rust，刚才又说考虑 Go，有结论了吗？" |
| **决策悬空** | 决策属性没有 outcome 边 | "你说要考虑要不要招人，后来定了吗？" |

## 六、参与决策

不是所有偏差都要说。三级：

| 级别 | 条件 | 行为 |
|------|------|------|
| 🔴 **打断** | 偏差 > 0.8 且此时非活跃对话 | 直接说 |
| 🟡 **提醒** | 偏差 0.5-0.8 | 等对话自然间隙说 |
| 🟢 **归档** | 偏差 < 0.5 | 记下，不推 |

特别注意：用户跟我聊天时，聊天流活跃 → 我只提**高于 0.8 的打断级偏差**。其他记下，等对话结束再说。

---

## 七、实现路径

不依赖外部框架。基于 Hermes 现有能力：

- **Cronjob 触发**：每 15 分钟（可配）执行一次检查
- **想法图谱存储**：Memory 系统 + Obsidian 笔记
- **网络感知**：已有的 web_search / web_extract
- **对话流感知**：从 current session + memory 中提取
- **微信群/个人微信推送**：已有的 send_message
